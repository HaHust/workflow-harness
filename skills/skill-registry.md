## Skill Rules

- W01 must create a `skill-bundle.md` for every worker or reviewer run.
- Canonical skill file resolution is `skills/<family>/<skill-name>.md` relative to Workflow Home. W01 must verify the file and write its concrete path into the bundle.
- A bundle is valid only when every required and selected optional skill has a concrete readable file, load order, and expected output.
- Agents must read the registry and required skill files before task work and record `Skill Files Read` plus `Skill Load Status` in their run artifacts.
- A skill name without a loaded file is not permission to use the skill; return `BLOCKED` with `SKILL_NOT_LOADED`.
- A skill not listed in the bundle is forbidden for that run.
- A skill listed under `Forbidden Skills` must not be used even if it appears in this registry.
- Skill output is reviewed through the host agent artifact and the review profile assigned by W01.
